# v41 core traceability

Source specification:

- `specs/tzap-format-revisedv41.md`
- Document version: `0.41 / 2026-05-24.5`

Scope:

- The matrix covers the documented supported `tzap-core` and CLI archive
  workflows.
- Unsupported workflow shapes are acceptable only when they reject with stable
  diagnostics and are documented in `public-docs/`.
- RootAuth authenticator algorithms are mapped in
  [signing-plugin-traceability.md](signing-plugin-traceability.md).

## Current matrix result

No implementation gap is recorded for the documented supported workflows in this
pass. Rows marked `Unsupported and documented` are intentionally outside the
current CLI/core surface and are covered by stable rejection behavior plus
`public-docs/` guidance.

The broader phrase "complete v41 compliance" still requires an external
conformance definition for optional or future profiles that the project does
not currently support.
