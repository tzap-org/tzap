# tzap v0.36 Conformance Matrix

Status: draft evidence table
Primary spec: `specs/tzap-format-revisedv36.md`, section 29

This matrix is the reviewer-facing map from every v0.36 conformance obligation
to implementation evidence. It is intentionally conservative: `complete` means
the current code and tests cover the implemented obligation shape; `partial`
means code exists but some required mode, scale case, mutation, or product
boundary still needs work; `unsupported` means the public API/CLI does not
offer that mode and should reject or avoid claiming it; `deferred` means the
feature is intentionally outside the current release claim.
